Proyecto Semestral - Entrega 3
Durante el semestre desarrollarán un proyecto que tendrá cuatro entregas progresivas. El proyecto será en parejas, y se describe a continuación la Entrega 3.



Objetivo
Aplicar los conocimientos adquiridos sobre Programación Orientada a Objetos, enfocado en los temas de Clases y Objetos, Herencia y Polimorfismo. Además incluyendo los temas de Manejo de Excepciones y Manejo de E/S avanzadas.



Descripción
Considerando la descripción de la Entrega N°1 y la Entrega N°2, realizar los siguientes cambios y mejoras a su programa.

Considerar las nuevas restricciones de combinaciones:

No se pueden combinar diferentes líquidos en un sólo estanque.
No se pueden combinar diferentes gases en un sólo estanque,

Requisitos
Se les entregará un archivo CSV con una lista aleatoria de productos a transportar al momento de evaluar.
El archivo tendrá el siguiente formato estándar:
ID_Producto
Nombre_Producto
Tipo
Normal (sin restricciones especiales de transporte)
Refrigerado
Inflamable
Masa
Sólida (contenedor)
Líquida (estanque)
Gas (estanque)
Peso
Peso del total de productos en Kilogramos
El programa deberá leer el archivo y distribuir la lista de productos de la forma más optima en relación a costo de transporte. (menor costo posible)
1 tonelada = 1.000 kilogramos.


Enunciado Entrega 3
Deberán adaptar el código para que la lista leída desde el CSV sea almacenada en una Base de Datos a libre elección. (8 pts)
Los cálculos posteriores se deben realizar leyendo la información desde la Base de Datos y NO desde el CSV.
Deberán adaptar el código para que los costos por cada tipo de vehículo sean editables. (8 pts)
Posterior a editar los costos, se debe recalcular la optimización nuevamente.
Deberán adaptar el código para que la información obtenida por interfaz gráfica especificada en la Entrega N°2 se pueda exportar a PDF. (8 pts)
2 décimas extras si también se puede exportar a CSV y JSON.
Puntaje total: 24 pts



Entrega (Domingo 13 de noviembre)

Solo un integrante debe subir la Entrega #3.

Debe crear un ZIP o RAR con todos los archivos y adjuntar a esta Entrega.

Debe crear un repositorio GIT y adjuntar el enlace a este (github/gitlab/etc) a este proyecto, mostrando claramente el historial de commits y cambios a medida que avanzan en el proyecto. (2 décimas extra)

Bonificación por utilizar un lenguaje diferente a Python (5 décimas extra)
